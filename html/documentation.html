1. Introduction

This document describes the XML-Encoding of an dictionary entry as used in the projekt Loanwords in Rabbinic Literature.

The XML-representation of an entry is realized in TEI in form of the customization TEI Lex-0. For further information see the respective documentation:

TEI Lex-0: Romary, Laurent; Tasovac, Toma. 2024, TEI Lex-0 — A baseline encoding for lexicographic data. Version: 0.9.3 (https://dariah-eric.github.io/lexicalresources/pages/TEILex0/TEILex0.html).

TEI Guidelines: TEI Consortium. 2024, TEI: Guidelines for Electronic Text Encoding and Interchange. P5 Version 4.8.0. Last updated on 2nd September 2024 (https://tei-c.org/release/doc/tei-p5-doc/en/html/index.html).

2. Structure of an entry

Each entry of the dictionary is a entry element in the body of a TEI document.

2.1 Basic structure

Every entry carries an unique id. The basic structure of the TEI documents is the following:


<TEI xmlns="http://www.tei-c.org/ns/1.0">
   <teiHeader>
      ...   
   </teiHeader>
   <text>
      <body>
         <entry xml:id="afes-dlgenr">
            ...
         </entry>
      </body>
   </text>
</TEI>

2.2 Subsections of an entry

The subelements of an entry are form, sense, etym, note and a ref element used for the bibliography.


<entry>
   <form> ... </form>
   <sense> ... </sense>
   <etym> ... </etym>
   <note> ... </note>
   <ref type="bibliography"> ... </ref>
</entry>

When a lemma has two or more strongly different meanings two or more sense elements are used. If the meaning differs only slightly they are noted in the same sense element separated by a semicolon.

3. The lemma

3.1 Encoding of the lemma

The lemma of an entry is encoded with a form element with the attribute type set to lemma. The lemma itself is embedded in an orth element that also contains an attribute xml:lang which specifies the given language (jpa: Jewish Palestinian Aramaic). For the lemma a transcription is given following the guidelines of the Frankfurter Judaistische Beiträge and the rabbinic pronounciation is noted. Furthermore grammatical information is provided (part of speech) with the help of the gram element nested inside of a gramGrp element.


<form type="lemma">
   <orth xml:lang="jpa">ביז</orth>
   <orth type="transliteration" xml:lang="jpa-x-frankfurt-fjb">byz</orth>
   <pron notation="rabbinic">biz</pron>
   <gramGrp>
      <gram type="pos">noun</gram>
      <gram type="gen">m.</gram>
   </gramGrp>
</form>

3.2 Encoding of variants of the lemma

Sometimes the lemma is written in different forms in the manuscripts. This information is encoded in a form element with the attribute type set to variant nested into the form element for the lemma (type = lemma). When more than one variant appears each gets its own form element.


<form type="lemma">
   ...
   <form type="variant">
      <orth xml:lang="jpa">הפס</orth>
      <orth type="transliteration" xml:lang="jpa-x-frankfurt-fjb">hfs</orth>
      <pron notation="rabbinic">hafes</pron>
   </form>
   ...
</form>

4. Derivates

Derived forms of a lemma are encoded within form elements beneath the entry level and as siblings of the form element of the lemma.


<entry>
    ...                    
    <form type="inflected">
        <orth xml:lang="jpa">ביזיא</orth>
        <orth type="transliteration" xml:lang="jpa-x-frankfurt-fjb">byzyʾ</orth>
        <pron notation="rabbinic">biziaʾ</pron>
        <gramGrp>
            <gram type="pos">noun</gram>
            <gram type="num">pl.</gram>
            <gram type="mood">det.</gram>
        </gramGrp>
    </form>
    ...
</entry>

5. Senses

The sense of a lemma gets the markup sense and is a subelement of the entry element. When a lemma has more than one sense multiple sense elements are used. Every sense element carries an id.


<entry>
    ...
    <sense xml:id="biz-dlgenr-1">
        ...
    </sense>
    <sense xml:id="biz-dlgenr-2">
        ...
    </sense>
    ...
</entry>

5.1 The structure of the sense element

The sense element has subelements for usage, the translation, the translation equivalent and for examples:


<sense xml:id="biz-dlgenr">
    <usg>...</usg>
    <cit type="translation">...</cit>
    <cit type="translationEquivalent">...</cit>
    <cit type="example">...</cit>
</sense>

5.2 The usage of a word

The usage of a lemma is encoded with the usg element. The different forms of usage are noted with the type attribute.


<sense>
    <usg type="domain">human life/biology</usg>
    <usg type="socioCultural" norm="borrowing">core borrowing</usg>
    <usg type="normativity">standard</usg>
    <usg type="meaningType">lit.</usg>
    <usg type="frequency">hapax legomenon</usg>
    ...
</sense>

5.3 The translation of a word

The translation of a lemma is given in a cit element with the attribute type set to translation. Furthermore, the language of the translation has to be noted. The translation itself is encoded with a quote element.


<sense>
    ...
    <cit type="translation" xml:lang="en">
         <quote>injustice</quote>
    </cit>
    ...
</sense>

5.4 The translation equivalent

Because in the Dictionary of Loandwords in Rabbinic Literature - as the name says - the lemmata are loanwords a translation equivalent must be noted. This can be done with the cit element with the type attribute set to translationEquivalent. The language from which the word is taken is specified with the lang element embedded in an usg element. The translation equivalent itself is encoded with a quote element.


<sense>
    ...
    <cit type="translationEquivalent" xml:lang="grc">
        <usg>
            <lang>Post-Classical Greek</lang>
        </usg>
        <quote>βυζίον</quote>
    </cit>
    ...
</sense>

5.5 Examples

Examples for the usage of a word are tagged as cit of the type example. The language of the example must be noted (xml:lang). The example itself is encoded via a quote element. The bibl element contains the source of the example within ref elements. The translation of the example follows in a cit element with type set to translation and with an embedded quote.


<sense>
    ...
    <cit type="example" xml:lang="jpa">
        <quote>כאינש דאמר לייט ביזה דהכדן איינק</quote>
        <bibl>
            <ref type="witness">GenR 5,9</ref>
            <ref type="witness">GenR I 39: 2</ref>
        </bibl>
        <cit type="translation" xml:lang="en">
            <quote>It is in fact as one might say: ‘Cursed be the breast that suckled such a one as this’. (Freedman / Simon I 40)</quote>
        </cit>
    </cit>
    ...
</sense>

6. Etymology

The etymology of a lemma is encoded in an etym element. The type attribute specifies the relation between the source and the target language. The lbl element contains information which should go into the presentation as it is. The lang element gives the name of the language the word or word part is taken from. The word or word part itself is tagged with a cit element. Instead of a quote for the etymon the xr element is used to encode a reference to another dictionary (Liddell-Scott-Jones).


<entry>
    ...
    <etym type="borrowing">
        <lbl>source language</lbl>
        <lang>Ancient Greek+</lang>
            <cit type="etymon" xml:lang="grc">
                <xr type="related">
                    <lbl>Cf. LSJ</lbl>
                    <ref target="http://www.perseus.tufts.edu/hopper/morph?l=bi%2Fa&la=greek&can=bi%2Fa0" type="entry">βία</ref>
                </xr>
           </cit>
    </etym>
    ...
</entry>

7. Note

A free text with a discussion about the entry is given within a note element.


<entry>
    ...
    <note>
        ...
    </note>
    ...
</entry>

Within a note the element term is used to tag words of a different language then English and for bibliographical information the bibl element is used with an attribute type set to ancient or modern.

8. Bibliography

Finally the bibliography is encoded with a ref element of type bibliography:


<entry>
    ...
    <ref type="bibliography">
        <bibl>
            <title>Sperber 1984</title>
            <citedRange>69</citedRange>
        </bibl>
        <bibl>
            <title>Sokoloff DJPA</title>
            <citedRange>95</citedRange>
        </bibl>
    </ref>
</entry>
